## 🔧 已解决的关键技术问题

### 问题1：评估列表NoneType错误
**现象**：`'NoneType' object has no attribute 'model_validate'`
**根本原因**：Pydantic无法自动序列化SQLAlchemy关联对象
**解决方案**：
1. 修改 `AbilityAssessmentOutSchema` 字段类型定义
2. 正确配置 `model_config = ConfigDict(from_attributes=True)`
3. 使用 `UserOutSchema` 和 `StudentOutSchema` 进行类型化序列化
4. 在service层添加防御性编程，过滤None值

### 问题2：教练下拉框显示为空
**现象**：课程/评估页面教练选择下拉框无数据
**原因分析**：
- 前端期望 `response.data.data.list` 但后端返回 `response.data.data.items`
- 未正确调用用户API获取全部教练列表
**解决方案**：
1. 修改前端数据访问路径：`list` → `items`
2. 集成 `UserAPI.listUser()` 获取系统所有用户
3. 添加状态筛选（只显示活跃用户）

### 问题3：新建评估保存失败
**现象**：`"AbilityAssessmentCreateSchema" object has no field "overall_score"`
**原因**：Pydantic验证器尝试设置 `overall_score` 但字段未声明
**解决方案**：
1. 在 `AbilityAssessmentCreateSchema` 中添加 `overall_score` 字段声明
2. 在 `@model_validator` 中正确计算综合评分
3. 确保字段验证顺序正确

### 问题4：学员姓名显示"未知"
**现象**：评估列表中学员姓名显示为"未知"而非真实姓名
**原因**：Pydantic `from_attributes=True` 无法自动处理SQLAlchemy关联
**解决方案**：
1. 在 `AbilityAssessmentOutSchema` 中添加 `student` 和 `coach` 字段
2. 使用正确的Schema类型引用：`StudentOutSchema` 和 `UserOutSchema`
3. 移除冗余的手动序列化代码

### 问题5：学员导入模板下载异常
**现象**：下载的Excel文件内容为 `[object Object]`
**原因**：前端错误处理Blob响应
**解决方案**：
1. 检查 `request.ts` 拦截器对 `responseType: "blob"` 的处理
2. 修复 `handleDownloadTemplate` 函数，正确提取 `response.data`
3. 确保Blob对象创建和文件下载流程正确
