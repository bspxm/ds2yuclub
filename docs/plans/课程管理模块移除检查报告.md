# 课程管理模块移除检查报告

## 背景

课程管理（course）模块已被班级管理（class/team）模块完全替代。本报告分析 course 模块涉及的代码、数据表及其与其他模块的关联关系，供评估是否移除。

---

## 一、涉及的数据表（2张）

| 表名 | 模型 | 说明 |
|---|---|---|
| `badminton_course` | `CourseModel` | 课程定义表（无学期概念） |
| `badminton_student_course` | `StudentCourseModel` | 学员-课程报名关联表 |

### `badminton_course` 的外键（引用其他表）
- `coach_id` → `sys_user.id`（SET NULL）
- `created_id` / `updated_id` → `sys_user.id`

### `badminton_student_course` 的外键
- `student_id` → `badminton_student.id`（CASCADE）
- `course_id` → `badminton_course.id`（CASCADE）

### 其他表对 course 的外键引用（级联影响）
- `badminton_leave_record.student_course_id` → `badminton_student_course.id`（**RESTRICT**）
- `sys_user` 通过 relationship `courses_as_coach` 引用 `CourseModel`

---

## 二、关键关联：请假模块（leave）依赖 course

**这是移除的最大障碍。** 请假模块 `LeaveRequestModel` 有一个硬外键：

```
badminton_leave_record.student_course_id → badminton_student_course.id (RESTRICT)
```

相关代码：
- `leave/model.py` — 定义了 `student_course_id` FK 和 `student_course` relationship
- `leave/schema.py` — `LeaveRequestCreateSchema` 包含 `course_id` 字段
- `leave/service.py` — 使用 `CourseCRUD` 查询课程信息
- `leave/crud.py` — 预加载 `["student", "course"]`
- 前端 `leave-request/index.vue` 和 `leave-request.ts` — 引用 `course_id`、`course_name`

## 四、新旧系统对比

| 功能 | course（旧） | class/team（新） |
|---|---|---|
| 班级定义 | `CourseModel` | `ClassModel` |
| 学期关联 | 无 | `semester_id` FK |
| 班级类型 | 课程内容类型（常规/私教/小组等） | 排班类型（FIXED/FLEXIBLE） |
| 教练分配 | `coach_id` | `coach_id` |
| 时间安排 | 简单字段 | `weekly_schedule` + 独立 `ClassScheduleModel` |
| 学员报名 | `StudentCourseModel` | `PurchaseModel`（含财务数据） |
| 出勤跟踪 | 通过报名表字段计数 | 独立 `ClassAttendanceModel`（丰富数据） |
| 请假管理 | `LeaveRequestModel` → `StudentCourseModel` | 出勤表的 `is_leave` 字段 |
| 财务管理 | 无 | `PurchaseModel` 含价格、折扣、结算 |
| 课时管理 | 无 | 购买/结转/信用课时完整生命周期 |

**结论**：class/team 系统完全覆盖了 course 的功能，且更完善。

---

## 五、风险评估

### 低风险
- 删除 course 模块本体（后端 4 文件 + 前端 2 文件）
- 移除 `__init__.py` 中的导出
- 移除 `enums.py` 中的 `CourseTypeEnum`
- 移除 `student/model.py` 中的 `course_enrollments` relationship
- 移除 `user/model.py` 中的 `courses_as_coach` relationship
- 移除菜单 SQL 中的课程条目

### 中风险（需重构请假模块）
- **请假模块（leave）是最大的关联依赖**，`LeaveRequestModel` 的 `student_course_id` FK 指向 `badminton_student_course` 表
- 移除 course 后，请假模块需要：
  - 移除或修改 `student_course_id` 外键（需数据库迁移）
  - 修改请假业务逻辑，不再关联课程
  - 修改前端请假页面的课程选择器

### 数据库操作
- 需先生成 Alembic 迁移脚本来处理表删除和 `leave_record` 表的外键变更
- 建议先备份数据库

---

## 六、建议操作步骤

如果决定移除，建议按以下顺序：

1. **先重构请假模块**：将 `leave_record` 表的 `student_course_id` FK 改为可选或移除，调整业务逻辑
2. **删除后端 course 模块**：删除 `course/` 目录，清理所有引用
3. **删除前端 course 模块**：删除页面和 API 文件，清理引用
4. **生成 Alembic 迁移**：处理 `badminton_course` 和 `badminton_student_course` 表的删除
5. **清理菜单/权限数据**：更新 SQL 脚本和数据库中的菜单配置
